Authority Boundary

The Electron main process is the Core Engine. It owns domain workflows, vault unlock state, store access, application workflow state, and validation of guide actions.

The renderer mirrors view state returned by main and keeps only ephemeral display state such as drafts, selected panels, chat input, animation state, loading state, and errors.

Migration Status

Main startup is behind src/main/api, domains self-register IPC routes, generic database snapshot routes have been removed, and architecture checks are in place.

Runtime Architecture

Components communicate through narrow contracts. Generic transport is allowed, but generic mutation channels and direct store access are not.

Electron Main

  • Application authority and Core Engine.
  • Owns domain workflows and in-memory state machines.
  • Reads and writes SQLite through domain repositories.
  • Starts and supervises the local guide agent sidecar.

Preload Bridge

  • Exposes a generic request/subscription bridge.
  • Forwards renderer requests over generic main IPC.
  • Returns unsubscribe functions for renderer subscriptions.
  • Does not know which domains or routes exist.

Renderer

  • Renders React UI, Pixi animation, and room interactions.
  • Calls renderer API clients instead of main internals.
  • Holds display-only state and local form drafts.
  • Never opens the store or talks to the agent directly.

Store Access

Store access flows service to repository interface to repository implementation to the generic SQLite adapter. Domain-specific SQL lives in the owning domain repository.

Agent Contract

The guide agent is advisory. It returns structured proposed actions, and main-process domain services validate every action before mutating state or publishing renderer updates.

State Logic

Session workflow state is a main-process domain contract. The graph defines legal movement, and UI controls must route workflow actions through graph-validated domain commands.

idleNo active workflow. Can start session or restore conservative state.
target_selectionSession entrypoint chooses the guide-prioritized active target.
preparationAssessment can be updated, approved, grounded, or closed.
stimulationStimulation may start, log sets, pause, ground, or begin closure.
interjectionPause or grounding space before continuing or closing.
closureClose down the session, request review, or return to work.
reviewReview outcome before closing the durable session record.
post_sessionReturn idle or begin another guided session.

Recovery Rule

  • No unfinished session restores idle.
  • Newest unfinished session with no stimulation sets restores preparation.
  • Newest unfinished session with stimulation sets restores interjection.

UI Rule

The renderer may hold selected panels and animation state, but it must not invent workflow transitions or directly skip from target selection to stimulation or from stimulation to ended history.

Guardrails

The architecture documents read as an enforcement contract: keep authority in main, keep display state in renderer, and keep persistent facts in repositories and migrations.

  • Main and renderer stay airgapped src/main/** has no React, Pixi, display state, or browser-only APIs. src/renderer/** imports no main internals.
  • Preload stays domain-agnostic Preload exposes generic request and subscription functions. It does not own route registration or domain endpoint lists.
  • Repositories own SQL Services and routes do not call database methods directly. Domain-specific SQL mappings live with the owning repository.
  • Schema changes are explicit Runtime app logic must not create, destroy, modify, or migrate schemas. Approved changes belong in migrations.
  • State machine validates workflow Human UI and future agent actions use the same graph-validated session flow commands.
  • Agent is advisory The agent proposes structured actions. Main-process services reject invalid actions before mutation.
  • Shared code is shape-only src/shared holds serializable contracts and must not become a domain behavior back door.
  • Local-only remains intact Electron continues blocking remote content, and durable app data stays in the encrypted local vault.
